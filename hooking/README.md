# Hooking (Telegram 자동화 - AutoHotkey)

이 스크립트는 **각 가상 데스크톱마다 텔레그램 창들이 동일한 크기/위치로 정렬되어 있는 상태**를 전제로, 감지된 텔레그램 창들에 대해 아래 작업을 자동으로 실행합니다.

1. **창 최신화**: 해당 창을 활성화한 뒤 `PgDn`을 짧게 여러 번 눌러(스크롤) 최신 상태로 만듭니다.
2. 화면에 보이는 **👍 공감** 클릭
3. 메시지 입력창 클릭 → 문구 입력 → Enter 전송

> 좌표 방식(상대좌표)이라서, 텔레그램 UI 배율/테마/창 내부 레이아웃이 바뀌면 클릭 위치가 약간 틀어질 수 있습니다.  
> 필요하면 `Hooking.ini` 값만 조절하면 됩니다.

## 준비물
- **AutoHotkey v2** 설치
- (선택) **가상화면 번호로 즉시 이동**하려면 `VirtualDesktopAccessor.dll`(또는 `vDesktop.dll`)을 `hooking` 폴더에 넣어주세요.
  - DLL이 없으면 “다중실행(선택 D, 메시지)”가 동작하지 않습니다.

## 실행
`Hooking.ahk` 더블클릭 실행

## 사용 팁
- “다중실행”은 D1~D10에서 선택된 가상화면들로 이동하며(번호로 즉시 이동) 각 화면의 감지된 Telegram 창들에 대해 실행합니다.
- 딜레이는 UI의 **다중 실행 딜레이(초)** 에서 직접 조절할 수 있습니다.
- 실행 중 중단: **F12**
- 상단의 **D1~D10**은 가상 데스크톱(CTRL+WIN+D) 선택입니다. 여러 개 선택 후 **다중실행(선택 D, 메시지)** 를 누르면 선택한 데스크톱들에만 메시지를 전송합니다(데스크톱 전환하며 실행).

## 설정(Hooking.ini)
- 창 최신화는 좌표가 필요 없습니다(키보드 PgDn 사용).
- `ThumbX/ThumbY` / `Thumb2X/Thumb2Y` / `Thumb3X/Thumb3Y`: 👍 클릭 좌표(클라이언트 기준). 실행 시 1~3 중 랜덤 1개를 클릭합니다.
- `InputX/InputDy`: 메시지 입력창을 클릭할 좌표(왼쪽에서 X, 아래에서 Dy)
- `DelayMinMs/DelayMaxMs`: 창/데스크톱 사이 딜레이(ms)
- 좌표는 **좌표찍기** 후 자동 저장됩니다.
- 메시지는 **메시지 저장**을 누르면 마지막 저장 파일이 다음 실행 때 자동으로 불러와집니다.
- 최근 저장/불러오기 했던 메시지 파일은 아래 “최근 메시지” 목록(최대 5개)에 표시되며, 목록을 클릭하면 파일 선택 창 없이 바로 불러옵니다.
